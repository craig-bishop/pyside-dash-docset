Detailed Description
--------------------



    The :class:`PySide.QtGui.QPainterPath` class provides a container for painting operations, enabling graphical shapes to be constructed and reused.

    A painter path is an object composed of a number of graphical building blocks, such as rectangles, ellipses, lines, and curves. Building blocks can be joined in closed subpaths, for example as a rectangle or an ellipse. A closed path has coinciding start and end points. Or they can exist independently as unclosed subpaths, such as lines and curves.

    A :class:`PySide.QtGui.QPainterPath` object can be used for filling, outlining, and clipping. To generate fillable outlines for a given painter path, use the :class:`PySide.QtGui.QPainterPathStroker` class. The main advantage of painter paths over normal drawing operations is that complex shapes only need to be created once; then they can be drawn many times using only calls to the :meth:`QPainter.drawPath()<PySide.QtGui.QPainter.drawPath>` function.

    :class:`PySide.QtGui.QPainterPath` provides a collection of functions that can be used to obtain information about the path and its elements. In addition it is possible to reverse the order of the elements using the :meth:`PySide.QtGui.QPainterPath.toReversed` function. There are also several functions to convert this painter path object into a polygon representation.

    .. _PySide.QtGui.QPainterPath_composing-a-qpainterpath:



Composing a QPainterPath
^^^^^^^^^^^^^^^^^^^^^^^^

    A :class:`PySide.QtGui.QPainterPath` object can be constructed as an empty path, with a given start point, or as a copy of another :class:`PySide.QtGui.QPainterPath` object. Once created, lines and curves can be added to the path using the :meth:`PySide.QtGui.QPainterPath.lineTo` , :meth:`PySide.QtGui.QPainterPath.arcTo` , :meth:`PySide.QtGui.QPainterPath.cubicTo` and :meth:`PySide.QtGui.QPainterPath.quadTo` functions. The lines and curves stretch from the :meth:`PySide.QtGui.QPainterPath.currentPosition` to the position passed as argument.

    The :meth:`PySide.QtGui.QPainterPath.currentPosition` of the :class:`PySide.QtGui.QPainterPath` object is always the end position of the last subpath that was added (or the initial start point). Use the :meth:`PySide.QtGui.QPainterPath.moveTo` function to move the :meth:`PySide.QtGui.QPainterPath.currentPosition` without adding a component. The :meth:`PySide.QtGui.QPainterPath.moveTo` function implicitly starts a new subpath, and closes the previous one. Another way of starting a new subpath is to call the :meth:`PySide.QtGui.QPainterPath.closeSubpath` function which closes the current path by adding a line from the :meth:`PySide.QtGui.QPainterPath.currentPosition` back to the path's start position. Note that the new path will have (0, 0) as its initial :meth:`PySide.QtGui.QPainterPath.currentPosition` .

    :class:`PySide.QtGui.QPainterPath` class also provides several convenience functions to add closed subpaths to a painter path: :meth:`PySide.QtGui.QPainterPath.addEllipse` , :meth:`PySide.QtGui.QPainterPath.addPath` , :meth:`PySide.QtGui.QPainterPath.addRect` , :meth:`PySide.QtGui.QPainterPath.addRegion` and :meth:`PySide.QtGui.QPainterPath.addText` . The :meth:`PySide.QtGui.QPainterPath.addPolygon` function adds an *unclosed* subpath. In fact, these functions are all collections of :meth:`PySide.QtGui.QPainterPath.moveTo` , :meth:`PySide.QtGui.QPainterPath.lineTo` and :meth:`PySide.QtGui.QPainterPath.cubicTo` operations.

    In addition, a path can be added to the current path using the :meth:`PySide.QtGui.QPainterPath.connectPath` function. But note that this function will connect the last element of the current path to the first element of given one by adding a line.

    Below is a code snippet that shows how a :class:`PySide.QtGui.QPainterPath` object can be used:

    The painter path is initially empty when constructed. We first add a rectangle, which is a closed subpath. Then we add two bezier curves which together form a closed subpath even though they are not closed individually. Finally we draw the entire path. The path is filled using the default fill rule, :attr:`Qt.OddEvenFill<Qt.FillRule>` . Qt provides two methods for filling paths:

QPainterPath Information
^^^^^^^^^^^^^^^^^^^^^^^^

    The :class:`PySide.QtGui.QPainterPath` class provides a collection of functions that returns information about the path and its elements.

    The :meth:`PySide.QtGui.QPainterPath.currentPosition` function returns the end point of the last subpath that was added (or the initial start point). The :meth:`PySide.QtGui.QPainterPath.elementAt` function can be used to retrieve the various subpath elements, the *number* of elements can be retrieved using the :meth:`PySide.QtGui.QPainterPath.elementCount` function, and the :meth:`PySide.QtGui.QPainterPath.isEmpty` function tells whether this :class:`PySide.QtGui.QPainterPath` object contains any elements at all.

    The :meth:`PySide.QtGui.QPainterPath.controlPointRect` function returns the rectangle containing all the points and control points in this path. This function is significantly faster to compute than the exact :meth:`PySide.QtGui.QPainterPath.boundingRect` which returns the bounding rectangle of this painter path with floating point precision.

    Finally, :class:`PySide.QtGui.QPainterPath` provides the :meth:`PySide.QtGui.QPainterPath.contains` function which can be used to determine whether a given point or rectangle is inside the path, and the :meth:`PySide.QtGui.QPainterPath.intersects` function which determines if any of the points inside a given rectangle also are inside this path.

    .. _PySide.QtGui.QPainterPath_qpainterpath-conversion:



QPainterPath Conversion
^^^^^^^^^^^^^^^^^^^^^^^

    For compatibility reasons, it might be required to simplify the representation of a painter path: :class:`PySide.QtGui.QPainterPath` provides the :meth:`PySide.QtGui.QPainterPath.toFillPolygon` , :meth:`PySide.QtGui.QPainterPath.toFillPolygons` and :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons` functions which convert the painter path into a polygon. The :meth:`PySide.QtGui.QPainterPath.toFillPolygon` returns the painter path as one single polygon, while the two latter functions return a list of polygons.

    The :meth:`PySide.QtGui.QPainterPath.toFillPolygons` and :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons` functions are provided because it is usually faster to draw several small polygons than to draw one large polygon, even though the total number of points drawn is the same. The difference between the two is the *number* of polygons they return: The :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons` creates one polygon for each subpath regardless of intersecting subpaths (i.e. overlapping bounding rectangles), while the :meth:`PySide.QtGui.QPainterPath.toFillPolygons` functions creates only one polygon for overlapping subpaths.

    The :meth:`PySide.QtGui.QPainterPath.toFillPolygon` and :meth:`PySide.QtGui.QPainterPath.toFillPolygons` functions first convert all the subpaths to polygons, then uses a rewinding technique to make sure that overlapping subpaths can be filled using the correct fill rule. Note that rewinding inserts additional lines in the polygon so the outline of the fill polygon does not match the outline of the path.

.. method:: PySide.QtGui.QPainterPath.angleAtPercent(t)


    :param t: :class:`PySide.QtCore.qreal`
    :rtype: :class:`PySide.QtCore.qreal`




    Returns the angle of the path tangent at the percentage ``t`` . The argument ``t`` has to be between 0 and 1.

    Positive values for the angles mean counter-clockwise while negative values mean the clockwise direction. Zero degrees is at the 3 o'clock position.

    Note that similarly to the other percent methods, the percentage measurement is not linear with regards to the length if curves are present in the path. When curves are present the percentage argument is mapped to the t parameter of the Bezier equations.

.. method:: PySide.QtGui.QPainterPath.toFillPolygon(matrix)


    :param matrix: :class:`PySide.QtGui.QTransform`
    :rtype: :class:`PySide.QtGui.QPolygonF`




    Converts the path into a polygon using the :class:`PySide.QtGui.QTransform` ``matrix`` , and returns the polygon.

    The polygon is created by first converting all subpaths to polygons, then using a rewinding technique to make sure that overlapping subpaths can be filled using the correct fill rule.

    Note that rewinding inserts addition lines in the polygon so the outline of the fill polygon does not match the outline of the path.

    .. seealso::  :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons`  :meth:`PySide.QtGui.QPainterPath.toFillPolygons`  :class:`QPainterPath Conversion<~PySide.QtGui.QPainterPath#QPainterPath Conversion>`

.. method:: PySide.QtGui.QPainterPath.toFillPolygons(matrix)


    :param matrix: :class:`PySide.QtGui.QTransform`
    :rtype: 




    Converts the path into a list of polygons using the :class:`PySide.QtGui.QTransform` ``matrix`` , and returns the list.

    The function differs from the :meth:`PySide.QtGui.QPainterPath.toFillPolygon` function in that it creates several polygons. It is provided because it is usually faster to draw several small polygons than to draw one large polygon, even though the total number of points drawn is the same.

    The :meth:`PySide.QtGui.QPainterPath.toFillPolygons` function differs from the :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons` function in that it create only polygon for subpaths that have overlapping bounding rectangles.

    Like the :meth:`PySide.QtGui.QPainterPath.toFillPolygon` function, this function uses a rewinding technique to make sure that overlapping subpaths can be filled using the correct fill rule. Note that rewinding inserts addition lines in the polygons so the outline of the fill polygon does not match the outline of the path.

    .. seealso::  :meth:`PySide.QtGui.QPainterPath.toSubpathPolygons`  :meth:`PySide.QtGui.QPainterPath.toFillPolygon`  :class:`QPainterPath Conversion<~PySide.QtGui.QPainterPath#QPainterPath Conversion>`
