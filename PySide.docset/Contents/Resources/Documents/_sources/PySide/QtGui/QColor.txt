Detailed Description
--------------------



    The :class:`PySide.QtGui.QColor` class provides colors based on RGB, HSV or CMYK values.

    A color is normally specified in terms of RGB (red, green, and blue) components, but it is also possible to specify it in terms of HSV (hue, saturation, and value) and CMYK (cyan, magenta, yellow and black) components. In addition a color can be specified using a color name. The color name can be any of the SVG 1.0 color names.

    +----------------------------------------------------------------------------------------------------+----------------------------------------------------------------------------------------------------+-----------------------------------------------------------------------------------------------------+
    |.. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-rgb.png|.. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-hsv.png|.. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-cmyk.png|
    +====================================================================================================+====================================================================================================+=====================================================================================================+
    |RGB                                                                                                 |HSV                                                                                                 |CMYK                                                                                                 |
    +----------------------------------------------------------------------------------------------------+----------------------------------------------------------------------------------------------------+-----------------------------------------------------------------------------------------------------+

    The :class:`PySide.QtGui.QColor` constructor creates the color based on RGB values. To create a :class:`PySide.QtGui.QColor` based on either HSV or CMYK values, use the :meth:`PySide.QtGui.QColor.toHsv` and :meth:`PySide.QtGui.QColor.toCmyk` functions respectively. These functions return a copy of the color using the desired format. In addition the static :meth:`PySide.QtGui.QColor.fromRgb` , :meth:`PySide.QtGui.QColor.fromHsv` and :meth:`PySide.QtGui.QColor.fromCmyk` functions create colors from the specified values. Alternatively, a color can be converted to any of the three formats using the :meth:`PySide.QtGui.QColor.convertTo` function (returning a copy of the color in the desired format), or any of the :meth:`PySide.QtGui.QColor.setRgb` , :meth:`PySide.QtGui.QColor.setHsv` and :meth:`PySide.QtGui.QColor.setCmyk` functions altering *this* color's format. The :meth:`PySide.QtGui.QColor.spec` function tells how the color was specified.

    A color can be set by passing an RGB string (such as "#112233"), or a color name (such as "blue"), to the :meth:`PySide.QtGui.QColor.setNamedColor` function. The color names are taken from the SVG 1.0 color names. The :meth:`PySide.QtGui.QColor.name` function returns the name of the color in the format "#RRGGBB". Colors can also be set using :meth:`PySide.QtGui.QColor.setRgb` , :meth:`PySide.QtGui.QColor.setHsv` and :meth:`PySide.QtGui.QColor.setCmyk` . To get a lighter or darker color use the :meth:`PySide.QtGui.QColor.lighter` and :meth:`PySide.QtGui.QColor.darker` functions respectively.

    The :meth:`PySide.QtGui.QColor.isValid` function indicates whether a :class:`PySide.QtGui.QColor` is legal at all. For example, a RGB color with RGB values out of range is illegal. For performance reasons, :class:`PySide.QtGui.QColor` mostly disregards illegal colors, and for that reason, the result of using an invalid color is undefined.

    The color components can be retrieved individually, e.g with :meth:`PySide.QtGui.QColor.red` , :meth:`PySide.QtGui.QColor.hue` and :meth:`PySide.QtGui.QColor.cyan` . The values of the color components can also be retrieved in one go using the :meth:`PySide.QtGui.QColor.getRgb` , :meth:`PySide.QtGui.QColor.getHsv` and :meth:`PySide.QtGui.QColor.getCmyk` functions. Using the RGB color model, the color components can in addition be accessed with :meth:`PySide.QtGui.QColor.rgb` .

    There are several related non-members: :class:`~PySide.QtGui.QRgb` is a typdef for an unsigned int representing the RGB value triplet (r, g, b). Note that it also can hold a value for the alpha-channel (for more information, see the :class:`Alpha-Blended Drawing<~PySide.QtGui.QColor#Alpha-Blended Drawing>` section). The :meth:`~PySide.QtGui.QColor.qRed` , :meth:`~PySide.QtGui.QColor.qBlue` and :meth:`~PySide.QtGui.QColor.qGreen` functions return the respective component of the given :class:`~PySide.QtGui.QRgb` value, while the :meth:`~PySide.QtGui.QColor.qRgb` and :meth:`~PySide.QtGui.QColor.qRgba` functions create and return the :class:`~PySide.QtGui.QRgb` triplet based on the given component values. Finally, the :meth:`~PySide.QtGui.QColor.qAlpha` function returns the alpha component of the provided :class:`~PySide.QtGui.QRgb` , and the :meth:`~PySide.QtGui.QColor.qGray` function calculates and return a gray value based on the given value.

    :class:`PySide.QtGui.QColor` is platform and device independent. The :class:`~PySide.QtGui.QColormap` class maps the color to the hardware.

    For more information about painting in general, see the :ref:`Paint System<paintsystem.html>` documentation.

Integer vs. Floating Point Precision
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

    :class:`PySide.QtGui.QColor` supports floating point precision and provides floating point versions of all the color components functions, e.g. :meth:`PySide.QtGui.QColor.getRgbF` , :meth:`PySide.QtGui.QColor.hueF` and :meth:`PySide.QtGui.QColor.fromCmykF` . Note that since the components are stored using 16-bit integers, there might be minor deviations between the values set using, for example, :meth:`PySide.QtGui.QColor.setRgbF` and the values returned by the :meth:`PySide.QtGui.QColor.getRgbF` function due to rounding.

    While the integer based functions take values in the range 0-255 (except :meth:`PySide.QtGui.QColor.hue` which must have values within the range 0-359), the floating point functions accept values in the range 0.0 - 1.0.

    .. _PySide.QtGui.QColor_alpha-blended-drawing:



Alpha-Blended Drawing
^^^^^^^^^^^^^^^^^^^^^

    :class:`PySide.QtGui.QColor` also support alpha-blended outlining and filling. The alpha channel of a color specifies the transparency effect, 0 represents a fully transparent color, while 255 represents a fully opaque color. For example:

    ::

        # Specify semi-transparent red
        painter.setBrush(QColor(255, 0, 0, 127))
        painter.drawRect(0, 0, self.width()/2, self.height())

        # Specify semi-transparent blue
        painter.setBrush(QColor(0, 0, 255, 127))
        painter.drawRect(0, 0, self.width(), self.height()/2)


    The code above produces the following output:

    .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/alphafill.png

    Alpha-blended drawing is supported on Windows, Mac OS X, and on X11 systems that have the X Render extension installed.

    The alpha channel of a color can be retrieved and set using the :meth:`PySide.QtGui.QColor.alpha` and :meth:`PySide.QtGui.QColor.setAlpha` functions if its value is an integer, and :meth:`PySide.QtGui.QColor.alphaF` and :meth:`PySide.QtGui.QColor.setAlphaF` if its value is qreal (double). By default, the alpha-channel is set to 255 (opaque). To retrieve and set *all* the RGB color components (including the alpha-channel) in one go, use the :meth:`PySide.QtGui.QColor.rgba` and :meth:`PySide.QtGui.QColor.setRgba` functions.

    .. _PySide.QtGui.QColor_predefined-colors:



Predefined Colors
^^^^^^^^^^^^^^^^^

    There are 20 predefined QColors described by the :attr:`Qt.GlobalColor` enum, including black, white, primary and secondary colors, darker versions of these colors and three shades of gray. :class:`PySide.QtGui.QColor` also recognizes a variety of color names; the static :meth:`PySide.QtGui.QColor.colorNames` function returns a :class:`PySide.QtCore.QStringList` color names that :class:`PySide.QtGui.QColor` knows about.

    .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qt-colors.png

    Additionally, the :attr:`Qt.color0<Qt.GlobalColor>` , :attr:`Qt.color1<Qt.GlobalColor>` and :attr:`Qt.transparent<Qt.GlobalColor>` colors are used for special purposes.

    :attr:`Qt.color0<Qt.GlobalColor>` (zero pixel value) and :attr:`Qt.color1<Qt.GlobalColor>` (non-zero pixel value) are special colors for drawing in QBitmaps. Painting with :attr:`Qt.color0<Qt.GlobalColor>` sets the bitmap bits to 0 (transparent; i.e., background), and painting with :attr:`Qt.color1<Qt.GlobalColor>` sets the bits to 1 (opaque; i.e., foreground).

    :attr:`Qt.transparent<Qt.GlobalColor>` is used to indicate a transparent pixel. When painting with this value, a pixel value will be used that is appropriate for the underlying pixel format in use.

The HSV Color Model
^^^^^^^^^^^^^^^^^^^

    The RGB model is hardware-oriented. Its representation is close to what most monitors show. In contrast, HSV represents color in a way more suited to the human perception of color. For example, the relationships "stronger than", "darker than", and "the opposite of" are easily expressed in HSV but are much harder to express in RGB.

    HSV, like RGB, has three components:


    * H, for hue, is in the range 0 to 359 if the color is chromatic (not gray), or meaningless if it is gray. It represents degrees on the color wheel familiar to most people. Red is 0 (degrees), green is 120, and blue is 240.
      
      .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-hue.png
    * S, for saturation, is in the range 0 to 255, and the bigger it is, the stronger the color is. Grayish colors have saturation near 0; very strong colors have saturation near 255.
      
      .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-saturation.png
    * V, for value, is in the range 0 to 255 and represents lightness or brightness of the color. 0 is black; 255 is as far from black as possible.
      
      .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qcolor-value.png

    Here are some examples: pure red is H=0, S=255, V=255; a dark red, moving slightly towards the magenta, could be H=350 (equivalent to -10), S=255, V=180; a grayish light red could have H about 0 (say 350-359 or 0-10), S about 50-100, and S=255.

    Qt returns a hue value of -1 for achromatic colors. If you pass a hue value that is too large, Qt forces it into range. Hue 360 or 720 is treated as 0; hue 540 is treated as 180.

    In addition to the standard HSV model, Qt provides an alpha-channel to feature :class:`alpha-blended drawing<~PySide.QtGui.QColor#Alpha-Blended Drawing>` .

The CMYK Color Model
^^^^^^^^^^^^^^^^^^^^

    While the RGB and HSV color models are used for display on computer monitors, the CMYK model is used in the four-color printing process of printing presses and some hard-copy devices.

    CMYK has four components, all in the range 0-255: cyan (C), magenta (M), yellow (Y) and black (K). Cyan, magenta and yellow are called subtractive colors; the CMYK color model creates color by starting with a white surface and then subtracting color by applying the appropriate components. While combining cyan, magenta and yellow gives the color black, subtracting one or more will yield any other color. When combined in various percentages, these three colors can create the entire spectrum of colors.

    Mixing 100 percent of cyan, magenta and yellow *does* produce black, but the result is unsatisfactory since it wastes ink, increases drying time, and gives a muddy colour when printing. For that reason, black is added in professional printing to provide a solid black tone; hence the term 'four color process'.

    In addition to the standard CMYK model, Qt provides an alpha-channel to feature :class:`alpha-blended drawing<~PySide.QtGui.QColor#Alpha-Blended Drawing>` .

.. method:: PySide.QtGui.QColor.darker([f=200])


    :param f: :class:`PySide.QtCore.int`
    :rtype: :class:`PySide.QtGui.QColor`




    Returns a darker (or lighter) color, but does not change this object.

    If the ``factor`` is greater than 100, this functions returns a darker color. Setting ``factor`` to 300 returns a color that has one-third the brightness. If the ``factor`` is less than 100, the return color is lighter, but we recommend using the :meth:`PySide.QtGui.QColor.lighter` function for this purpose. If the ``factor`` is 0 or negative, the return value is unspecified.

    The function converts the current RGB color to HSV, divides the value (V) component by ``factor`` and converts the color back to RGB.

.. method:: PySide.QtGui.QColor.setNamedColor(name)


    :param name: unicode




    Sets the RGB value of this :class:`PySide.QtGui.QColor` to ``name`` , which may be in one of these formats:


    * #RGB (each of R, G, and B is a single hex digit)
    * #RRGGBB
    * #RRRGGGBBB
    * #RRRRGGGGBBBB
    * A name from the list of colors defined in the list of :ref:`SVG color keyword names<http://www.w3.org/TR/SVG/types.html#ColorKeywords>` provided by the World Wide Web Consortium; for example, "steelblue" or "gainsboro". These color names work on all platforms. Note that these color names are *not* the same as defined by the :attr:`Qt.GlobalColor` enums, e.g. "green" and :attr:`Qt.green<Qt.GlobalColor>` does not refer to the same color.
    * ``transparent`` - representing the absence of a color.
    * *X11 only*: If :meth:`PySide.QtGui.QColor.allowX11ColorNames` returns true, any valid X11 color name. See the documentation for ``XParseColor()`` for information about valid X11 color names.

    The color is invalid if ``name`` cannot be parsed.

    .. seealso::  :meth:`PySide.QtGui.QColor.QColor`  :meth:`PySide.QtGui.QColor.name`  :meth:`PySide.QtGui.QColor.isValid`  :meth:`PySide.QtGui.QColor.allowX11ColorNames`
