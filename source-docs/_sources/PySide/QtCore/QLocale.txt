Detailed Description
--------------------



    The :class:`PySide.QtCore.QLocale` class converts between numbers and their string representations in various languages.

    :class:`PySide.QtCore.QLocale` is initialized with a language/country pair in its constructor and offers number-to-string and string-to-number conversion functions similar to those in :class:`PySide.QtCore.QString` .

    Example:

    ::

        egyptian = QLocale(QLocale.Arabic, QLocale.Egypt)
        s1 = egyptian.toString(1.571429E+07, 'e')
        s2 = egyptian.toString(10)

        (d, ok) = egyptian.toDouble(s1)
        (i, ok) = egyptian.toInt(s2)


    :class:`PySide.QtCore.QLocale` supports the concept of a default locale, which is determined from the system's locale settings at application startup. The default locale can be changed by calling the static member :meth:`PySide.QtCore.QLocale.setDefault` . Setting the default locale has the following effects:


    * If a :class:`PySide.QtCore.QLocale` object is constructed with the default constructor, it will use the default locale's settings.
    * :meth:`QString.toInt` , :meth:`QString.toDouble` , etc., interpret the string according to the default locale. If this fails, it falls back on the "C" locale.
    * :meth:`QString.arg` uses the default locale to format a number when its position specifier in the format string contains an 'L', e.g. "%L1".

    The following example illustrates how to use :class:`PySide.QtCore.QLocale` directly:

    ::

        QLocale.setDefault(QLocale(QLocale.Hebrew, QLocale.Israel))
        hebrew = QLocale() # Constructs a default QLocale
        s1 = hebrew.toString(15714.3, 'e')

        QLocale.setDefault(QLocale(QLocale.C))
        c = QLocale()
        (d, ok) = c.toDouble("1234,56")   # ok == false
        (d, ok) = c.toDouble("1234.56")   # ok == true, d == 1234.56

        QLocale.setDefault(QLocale(QLocale.German))
        german = QLocale()
        (d, ok) = german.toDouble("1234,56")   # ok == true, d == 1234.56
        (d, ok) = german.toDouble("1234.56")   # ok == true, d == 1234.56

        QLocale.setDefault(QLocale(QLocale.English, QLocale.UnitedStates))
        english = QLocale()
        string = '%s %s %10x' % (12345, english.toString(12345), 12345)
        # string == "12345 12,345 3039"


    When a language/country pair is specified in the constructor, one of three things can happen:


    * If the language/country pair is found in the database, it is used.
    * If the language is found but the country is not, or if the country is ``AnyCountry``, the language is used with the most appropriate available country (for example, Germany for German),
    * If neither the language nor the country are found, :class:`PySide.QtCore.QLocale` defaults to the default locale (see :meth:`PySide.QtCore.QLocale.setDefault` ).

    Use :meth:`PySide.QtCore.QLocale.language` and :meth:`PySide.QtCore.QLocale.country` to determine the actual language and country values used.

    An alternative method for constructing a :class:`PySide.QtCore.QLocale` object is by specifying the locale name.

    ::

        korean = QLocale("ko")
        swiss = QLocale("de_CH")


    This constructor converts the locale name to a language/country pair; it does not use the system locale database.

    :class:`PySide.QtCore.QLocale` 's data is based on Common Locale Data Repository v1.8.1.

.. class:: QLocale()
           QLocale(language[, country=AnyCountry])
           QLocale(other)
           QLocale(name)

    :param country: :attr:`PySide.QtCore.QLocale.Country`
    :param other: :class:`PySide.QtCore.QLocale`
    :param language: :attr:`PySide.QtCore.QLocale.Language`
    :param name: unicode

    Constructs a :class:`PySide.QtCore.QLocale` object initialized with the default locale. If no default locale was set using setDefaultLocale(), this locale will be the same as the one returned by :meth:`PySide.QtCore.QLocale.system` .

    .. seealso::  :meth:`PySide.QtCore.QLocale.setDefault` 

    Constructs a :class:`PySide.QtCore.QLocale` object with the specified ``language`` and ``country`` .


    * If the language/country pair is found in the database, it is used.
    * If the language is found but the country is not, or if the country is ``AnyCountry``, the language is used with the most appropriate available country (for example, Germany for German),
    * If neither the language nor the country are found, :class:`PySide.QtCore.QLocale` defaults to the default locale (see :meth:`PySide.QtCore.QLocale.setDefault` ).

    The language and country that are actually used can be queried using :meth:`PySide.QtCore.QLocale.language` and :meth:`PySide.QtCore.QLocale.country` .

    .. seealso::  :meth:`PySide.QtCore.QLocale.setDefault`  :meth:`PySide.QtCore.QLocale.language`  :meth:`PySide.QtCore.QLocale.country` 

    Constructs a :class:`PySide.QtCore.QLocale` object as a copy of ``other`` .


    Constructs a :class:`PySide.QtCore.QLocale` object with the specified ``name`` , which has the format "language[\_country][.codeset][@modifier]" or "C", where:


    * language is a lowercase, two-letter, ISO 639 language code,
    * territory is an uppercase, two-letter, ISO 3166 country code,
    * and codeset and modifier are ignored.

    If the string violates the locale format, or language is not a valid ISO 369 code, the "C" locale is used instead. If country is not present, or is not a valid ISO 3166 code, the most appropriate country is chosen for the specified language.

    The language and country codes are converted to their respective ``Language`` and ``Country`` enums. After this conversion is performed the constructor behaves exactly like :class:`PySide.QtCore.QLocale` (Country, Language).

    This constructor is much slower than :class:`PySide.QtCore.QLocale` (Country, Language).

.. attribute:: PySide.QtCore.QLocale.NumberOption

    This enum defines a set of options for number-to-string and string-to-number conversions. They can be retrieved with :meth:`PySide.QtCore.QLocale.numberOptions` and set with :meth:`PySide.QtCore.QLocale.setNumberOptions` .

    +----------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
    |Constant                    |Description                                                                                                                                                                                      |
    +============================+=================================================================================================================================================================================================+
    |QLocale.OmitGroupSeparator  |If this option is set, the number-to-string functions will not insert group separators in their return values. The default is to insert group separators.                                        |
    +----------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
    |QLocale.RejectGroupSeparator|If this option is set, the string-to-number functions will fail if they encounter group separators in their input. The default is to accept numbers containing correctly placed group separators.|
    +----------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

.. method:: PySide.QtCore.QLocale.toDouble(s)


    :param s: unicode
    :rtype: (float, bool ok)




    Returns the double represented by the localized string ``s`` , or 0.0 if the conversion failed.

    If ``ok`` is not 0, reports failure by setting \*ok to false and success by setting \*ok to true.

    Unlike :meth:`QString.toDouble` , this function does not fall back to the "C" locale if the string cannot be interpreted in this locale.

    ::

        c = QLocale(QLocale.C)
        (d, ok) = c.toDouble( "1234.56" )  # ok == true, d == 1234.56
        (d, ok) = c.toDouble( "1,234.56" ) # ok == true, d == 1234.56
        (d, ok) = c.toDouble( "1234,56" )  # ok == false

        german = QLocale(QLocale.German)
        (d, ok) = german.toDouble( "1234,56" )  # ok == true, d == 1234.56
        (d, ok) = german.toDouble( "1.234,56" ) # ok == true, d == 1234.56
        (d, ok) = german.toDouble( "1234.56" )  # ok == false

        (d, ok) = german.toDouble( "1.234" )    # ok == true, d == 1234.0


    Notice that the last conversion returns 1234.0, because '.' is the thousands group separator in the German locale.

    This function ignores leading and trailing whitespace.

    .. seealso::  :meth:`PySide.QtCore.QLocale.toFloat`  :meth:`PySide.QtCore.QLocale.toInt`  :meth:`PySide.QtCore.QLocale.toString`

.. method:: PySide.QtCore.QLocale.toInt(s[, base=0])


    :param s: unicode
    :param base: :class:`PySide.QtCore.int`
    :rtype: (int, bool ok)




    Returns the int represented by the localized string ``s`` , using base ``base`` . If ``base`` is 0 the base is determined automatically using the following rules: If the string begins with "0x", it is assumed to be hexadecimal; if it begins with "0", it is assumed to be octal; otherwise it is assumed to be decimal.

    If ``ok`` is not 0, failure is reported by setting \*ok to false, and success by setting \*ok to true.
