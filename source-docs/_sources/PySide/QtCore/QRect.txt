Detailed Description
--------------------



    The :class:`PySide.QtCore.QRect` class defines a rectangle in the plane using integer precision.

    A rectangle is normally expressed as an upper-left corner and a size. The size (width and height) of a :class:`PySide.QtCore.QRect` is always equivalent to the mathematical rectangle that forms the basis for its rendering.

    A :class:`PySide.QtCore.QRect` can be constructed with a set of left, top, width and height integers, or from a :class:`PySide.QtCore.QPoint` and a :class:`PySide.QtCore.QSize` . The following code creates two identical rectangles.

    ::

        r1 = QRect(100, 200, 11, 16)
        r2 = QRect(QPoint(100, 200), QSize(11, 16))


    There is a third constructor that creates a :class:`PySide.QtCore.QRect` using the top-left and bottom-right coordinates, but we recommend that you avoid using it. The rationale is that for historical reasons the values returned by the :meth:`PySide.QtCore.QRect.bottom` and :meth:`PySide.QtCore.QRect.right` functions deviate from the true bottom-right corner of the rectangle.

    The :class:`PySide.QtCore.QRect` class provides a collection of functions that return the various rectangle coordinates, and enable manipulation of these. :class:`PySide.QtCore.QRect` also provide functions to move the rectangle relative to the various coordinates. In addition there is a :meth:`PySide.QtCore.QRect.moveTo` function that moves the rectangle, leaving its top left corner at the given coordinates. Alternatively, the :meth:`PySide.QtCore.QRect.translate` function moves the rectangle the given offset relative to the current position, and the :meth:`PySide.QtCore.QRect.translated` function returns a translated copy of this rectangle.

    The :meth:`PySide.QtCore.QRect.size` function returns the rectange's dimensions as a :class:`PySide.QtCore.QSize` . The dimensions can also be retrieved separately using the :meth:`PySide.QtCore.QRect.width` and :meth:`PySide.QtCore.QRect.height` functions. To manipulate the dimensions use the :meth:`PySide.QtCore.QRect.setSize` , :meth:`PySide.QtCore.QRect.setWidth` or :meth:`PySide.QtCore.QRect.setHeight` functions. Alternatively, the size can be changed by applying either of the functions setting the rectangle coordinates, for example, :meth:`PySide.QtCore.QRect.setBottom` or :meth:`PySide.QtCore.QRect.setRight` .

    The :meth:`PySide.QtCore.QRect.contains` function tells whether a given point is inside the rectangle or not, and the :meth:`PySide.QtCore.QRect.intersects` function returns true if this rectangle intersects with a given rectangle. The :class:`PySide.QtCore.QRect` class also provides the :meth:`PySide.QtCore.QRect.intersected` function which returns the intersection rectangle, and the :meth:`PySide.QtCore.QRect.united` function which returns the rectangle that encloses the given rectangle and this:

    The :meth:`PySide.QtCore.QRect.isEmpty` function returns true if :meth:`PySide.QtCore.QRect.left` > :meth:`PySide.QtCore.QRect.right` or :meth:`PySide.QtCore.QRect.top` > :meth:`PySide.QtCore.QRect.bottom` . A null rectangle ( :meth:`PySide.QtCore.QRect.isNull` == true) on the other hand, has both width and height set to 0.

    Note that due to the way :class:`PySide.QtCore.QRect` and :class:`PySide.QtCore.QRectF` are defined, an empty :class:`PySide.QtCore.QRect` is defined in essentially the same way as :class:`PySide.QtCore.QRectF` .

    Finally, :class:`PySide.QtCore.QRect` objects can be streamed as well as compared.

Rendering
^^^^^^^^^

    When using an :attr:`anti-aliased<QPainter.RenderHint>` painter, the boundary line of a :class:`PySide.QtCore.QRect` will be rendered symmetrically on both sides of the mathematical rectangle's boundary line. But when using an aliased painter (the default) other rules apply.

    Then, when rendering with a one pixel wide pen the :class:`PySide.QtCore.QRect` 's boundary line will be rendered to the right and below the mathematical rectangle's boundary line.

    When rendering with a two pixels wide pen the boundary line will be split in the middle by the mathematical rectangle. This will be the case whenever the pen is set to an even number of pixels, while rendering with a pen with an odd number of pixels, the spare pixel will be rendered to the right and below the mathematical rectangle as in the one pixel case.

Coordinates
^^^^^^^^^^^

    The :class:`PySide.QtCore.QRect` class provides a collection of functions that return the various rectangle coordinates, and enable manipulation of these. :class:`PySide.QtCore.QRect` also provide functions to move the rectangle relative to the various coordinates.

    For example the :meth:`PySide.QtCore.QRect.left` , :meth:`PySide.QtCore.QRect.setLeft` and :meth:`PySide.QtCore.QRect.moveLeft` functions as an example: :meth:`PySide.QtCore.QRect.left` returns the x-coordinate of the rectangle's left edge, :meth:`PySide.QtCore.QRect.setLeft` sets the left edge of the rectangle to the given x coordinate (it may change the width, but will never change the rectangle's right edge) and :meth:`PySide.QtCore.QRect.moveLeft` moves the entire rectangle horizontally, leaving the rectangle's left edge at the given x coordinate and its size unchanged.

    .. image:: ../../../../../../../src/qt-everywhere-opensource-src-4.7.2/doc/src/images/qrect-coordinates.png

    Note that for historical reasons the values returned by the :meth:`PySide.QtCore.QRect.bottom` and :meth:`PySide.QtCore.QRect.right` functions deviate from the true bottom-right corner of the rectangle: The :meth:`PySide.QtCore.QRect.right` function returns * :meth:`PySide.QtCore.QRect.left` *+ :meth:`PySide.QtCore.QRect.width` *- 1* and the :meth:`PySide.QtCore.QRect.bottom` function returns * :meth:`PySide.QtCore.QRect.top` *+ :meth:`PySide.QtCore.QRect.height` *- 1* . The same is the case for the point returned by the :meth:`PySide.QtCore.QRect.bottomRight` convenience function. In addition, the x and y coordinate of the :meth:`PySide.QtCore.QRect.topRight` and :meth:`PySide.QtCore.QRect.bottomLeft` functions, respectively, contain the same deviation from the true right and bottom edges.

    We recommend that you use :meth:`PySide.QtCore.QRect.x` + :meth:`PySide.QtCore.QRect.width` and :meth:`PySide.QtCore.QRect.y` + :meth:`PySide.QtCore.QRect.height` to find the true bottom-right corner, and avoid :meth:`PySide.QtCore.QRect.right` and :meth:`PySide.QtCore.QRect.bottom` . Another solution is to use :class:`PySide.QtCore.QRectF` : The :class:`PySide.QtCore.QRectF` class defines a rectangle in the plane using floating point accuracy for coordinates, and the :meth:`QRectF.right()<PySide.QtCore.QRectF.right>` and :meth:`QRectF.bottom()<PySide.QtCore.QRectF.bottom>` functions *do* return the right and bottom coordinates.

    It is also possible to add offsets to this rectangle's coordinates using the :meth:`PySide.QtCore.QRect.adjust` function, as well as retrieve a new rectangle based on adjustments of the original one using the :meth:`PySide.QtCore.QRect.adjusted` function. If either of the width and height is negative, use the :meth:`PySide.QtCore.QRect.normalized` function to retrieve a rectangle where the corners are swapped.

    In addition, :class:`PySide.QtCore.QRect` provides the :meth:`PySide.QtCore.QRect.getCoords` function which extracts the position of the rectangle's top-left and bottom-right corner, and the :meth:`PySide.QtCore.QRect.getRect` function which extracts the rectangle's top-left corner, width and height. Use the :meth:`PySide.QtCore.QRect.setCoords` and :meth:`PySide.QtCore.QRect.setRect` function to manipulate the rectangle's coordinates and dimensions in one go.
