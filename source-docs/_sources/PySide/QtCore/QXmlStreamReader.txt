Detailed Description
--------------------



    The :class:`PySide.QtCore.QXmlStreamReader` class provides a fast parser for reading well-formed XML via a simple streaming API.

    :class:`PySide.QtCore.QXmlStreamReader` is a faster and more convenient replacement for Qt's own SAX parser (see :class:`PySide.QtXml.QXmlSimpleReader` ). In some cases it might also be a faster and more convenient alternative for use in applications that would otherwise use a DOM tree (see :class:`PySide.QtXml.QDomDocument` ). :class:`PySide.QtCore.QXmlStreamReader` reads data either from a :class:`PySide.QtCore.QIODevice` (see :meth:`PySide.QtCore.QXmlStreamReader.setDevice` ), or from a raw :class:`PySide.QtCore.QByteArray` (see :meth:`PySide.QtCore.QXmlStreamReader.addData` ).

    Qt provides :class:`PySide.QtCore.QXmlStreamWriter` for writing XML.

    The basic concept of a stream reader is to report an XML document as a stream of tokens, similar to SAX. The main difference between :class:`PySide.QtCore.QXmlStreamReader` and SAX is *how* these XML tokens are reported. With SAX, the application must provide handlers (callback functions) that receive so-called XML *events* from the parser at the parser's convenience. With :class:`PySide.QtCore.QXmlStreamReader` , the application code itself drives the loop and pulls *tokens* from the reader, one after another, as it needs them. This is done by calling :meth:`PySide.QtCore.QXmlStreamReader.readNext` , where the reader reads from the input stream until it completes the next token, at which point it returns the :meth:`PySide.QtCore.QXmlStreamReader.tokenType` . A set of convenient functions including :meth:`PySide.QtCore.QXmlStreamReader.isStartElement` and :meth:`PySide.QtCore.QXmlStreamReader.text` can then be used to examine the token to obtain information about what has been read. The big advantage of this *pulling* approach is the possibility to build recursive descent parsers with it, meaning you can split your XML parsing code easily into different methods or classes. This makes it easy to keep track of the application's own state when parsing XML.

    A typical loop with :class:`PySide.QtCore.QXmlStreamReader` looks like this:

    ::

        xml = QXmlStreamReader()
        ...
        while not xml.atEnd():
            xml.readNext();
            ... # do processing

        if xml.hasError():
            ... # do error handling



    :class:`PySide.QtCore.QXmlStreamReader` is a well-formed XML 1.0 parser that does *not* include external parsed entities. As long as no error occurs, the application code can thus be assured that the data provided by the stream reader satisfies the W3C's criteria for well-formed XML. For example, you can be certain that all tags are indeed nested and closed properly, that references to internal entities have been replaced with the correct replacement text, and that attributes have been normalized or added according to the internal subset of the DTD.

    If an error occurs while parsing, :meth:`PySide.QtCore.QXmlStreamReader.atEnd` and :meth:`PySide.QtCore.QXmlStreamReader.hasError` return true, and :meth:`PySide.QtCore.QXmlStreamReader.error` returns the error that occurred. The functions :meth:`PySide.QtCore.QXmlStreamReader.errorString` , :meth:`PySide.QtCore.QXmlStreamReader.lineNumber` , :meth:`PySide.QtCore.QXmlStreamReader.columnNumber` , and :meth:`PySide.QtCore.QXmlStreamReader.characterOffset` are for constructing an appropriate error or warning message. To simplify application code, :class:`PySide.QtCore.QXmlStreamReader` contains a :meth:`PySide.QtCore.QXmlStreamReader.raiseError` mechanism that lets you raise custom errors that trigger the same error handling described.

    The :ref:`QXmlStream Bookmarks Example<xml/streambookmarks>` illustrates how to use the recursive descent technique to read an XML bookmark file (XBEL) with a stream reader.

    .. _PySide.QtCore.QXmlStreamReader_namespaces:



Namespaces
^^^^^^^^^^

    :class:`~PySide.QtCore.QXmlStream` understands and resolves XML namespaces. E.g. in case of a :attr:`StartElement<QXmlStreamReader.TokenType>` , :meth:`PySide.QtCore.QXmlStreamReader.namespaceUri` returns the namespace the element is in, and :meth:`PySide.QtCore.QXmlStreamReader.name` returns the element's *local* name. The combination of namespaceUri and name uniquely identifies an element. If a namespace prefix was not declared in the XML entities parsed by the reader, the namespaceUri is empty.

    If you parse XML data that does not utilize namespaces according to the XML specification or doesn't use namespaces at all, you can use the element's :meth:`PySide.QtCore.QXmlStreamReader.qualifiedName` instead. A qualified name is the element's :meth:`PySide.QtCore.QXmlStreamReader.prefix` followed by colon followed by the element's local :meth:`PySide.QtCore.QXmlStreamReader.name` - exactly like the element appears in the raw XML data. Since the mapping namespaceUri to prefix is neither unique nor universal, :meth:`PySide.QtCore.QXmlStreamReader.qualifiedName` should be avoided for namespace-compliant XML data.

    In order to parse standalone documents that do use undeclared namespace prefixes, you can turn off namespace processing completely with the :meth:`PySide.QtCore.QXmlStreamReader.namespaceProcessing` property.

    .. _PySide.QtCore.QXmlStreamReader_incremental-parsing:



Incremental parsing
^^^^^^^^^^^^^^^^^^^

    :class:`PySide.QtCore.QXmlStreamReader` is an incremental parser. It can handle the case where the document can't be parsed all at once because it arrives in chunks (e.g. from multiple files, or over a network connection). When the reader runs out of data before the complete document has been parsed, it reports a :attr:`PrematureEndOfDocumentError<QXmlStreamReader.Error>` . When more data arrives, either because of a call to :meth:`PySide.QtCore.QXmlStreamReader.addData` or because more data is available through the network :meth:`PySide.QtCore.QXmlStreamReader.device` , the reader recovers from the :attr:`PrematureEndOfDocumentError<QXmlStreamReader.Error>` error and continues parsing the new data with the next call to :meth:`PySide.QtCore.QXmlStreamReader.readNext` .

    For example, if your application reads data from the network using a :class:`network access manager<PySide.QtNetwork.QNetworkAccessManager>` , you would issue a :class:`network request<PySide.QtNetwork.QNetworkRequest>` to the manager and receive a :class:`network reply<PySide.QtNetwork.QNetworkReply>` in return. Since a :class:`PySide.QtNetwork.QNetworkReply` is a :class:`PySide.QtCore.QIODevice` , you connect its :meth:`PySide.QtNetwork.QNetworkReply.readyRead` signal to a custom slot, e.g. ``slotReadyRead()`` in the code snippet shown in the discussion for :class:`PySide.QtNetwork.QNetworkAccessManager` . In this slot, you read all available data with :meth:`PySide.QtNetwork.QNetworkReply.readAll` and pass it to the XML stream reader using :meth:`PySide.QtCore.QXmlStreamReader.addData` . Then you call your custom parsing function that reads the XML events from the reader.

    .. _PySide.QtCore.QXmlStreamReader_performance-and-memory-consumption:



Performance and memory consumption
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

    :class:`PySide.QtCore.QXmlStreamReader` is memory-conservative by design, since it doesn't store the entire XML document tree in memory, but only the current token at the time it is reported. In addition, :class:`PySide.QtCore.QXmlStreamReader` avoids the many small string allocations that it normally takes to map an XML document to a convenient and Qt-ish API. It does this by reporting all string data as :class:`PySide.QtCore.QStringRef` rather than real :class:`PySide.QtCore.QString` objects. :class:`PySide.QtCore.QStringRef` is a thin wrapper around :class:`PySide.QtCore.QString` substrings that provides a subset of the :class:`PySide.QtCore.QString` API without the memory allocation and reference-counting overhead. Calling :meth:`QStringRef.toString` on any of those objects returns an equivalent real :class:`PySide.QtCore.QString` object.

.. method:: PySide.QtCore.QXmlStreamReader.atEnd()


    :rtype: :class:`PySide.QtCore.bool`




    Returns true if the reader has read until the end of the XML document, or if an :meth:`PySide.QtCore.QXmlStreamReader.error` has occurred and reading has been aborted. Otherwise, it returns false.

    When :meth:`PySide.QtCore.QXmlStreamReader.atEnd` and :meth:`PySide.QtCore.QXmlStreamReader.hasError` return true and :meth:`PySide.QtCore.QXmlStreamReader.error` returns :attr:`PrematureEndOfDocumentError<QXmlStreamReader.Error>` , it means the XML has been well-formed so far, but a complete XML document has not been parsed. The next chunk of XML can be added with :meth:`PySide.QtCore.QXmlStreamReader.addData` , if the XML is being read from a :class:`PySide.QtCore.QByteArray` , or by waiting for more data to arrive if the XML is being read from a :class:`PySide.QtCore.QIODevice` . Either way, :meth:`PySide.QtCore.QXmlStreamReader.atEnd` will return false once more data is available.

.. method:: PySide.QtCore.QXmlStreamReader.readElementText(behaviour)


    :param behaviour: :attr:`PySide.QtCore.QXmlStreamReader.ReadElementTextBehaviour`
    :rtype: unicode




    Convenience function to be called in case a :attr:`StartElement<QXmlStreamReader.TokenType>` was read. Reads until the corresponding :attr:`EndElement<QXmlStreamReader.TokenType>` and returns all text in-between. In case of no error, the current token (see :meth:`PySide.QtCore.QXmlStreamReader.tokenType` ) after having called this function is :attr:`EndElement<QXmlStreamReader.TokenType>` .

    The function concatenates :meth:`PySide.QtCore.QXmlStreamReader.text` when it reads either :attr:`Characters<QXmlStreamReader.TokenType>` or :attr:`EntityReference<QXmlStreamReader.TokenType>` tokens, but skips :attr:`ProcessingInstruction<QXmlStreamReader.TokenType>` and :attr:`Comment<QXmlStreamReader.TokenType>` . If the current token is not :attr:`StartElement<QXmlStreamReader.TokenType>` , an empty string is returned.

    The ``behaviour`` defines what happens in case anything else is read before reaching :attr:`EndElement<QXmlStreamReader.TokenType>` . The function can include the text from child elements (useful for example for HTML), ignore child elements, or raise an :attr:`UnexpectedElementError<QXmlStreamReader.Error>` and return what was read so far.



.. method:: PySide.QtCore.QXmlStreamReader.readElementText()


    :rtype: unicode




    This function overloads :meth:`PySide.QtCore.QXmlStreamReader.readElementText` .

    Calling this function is equivalent to calling readElementText( :attr:`ErrorOnUnexpectedElement<QXmlStreamReader.ReadElementTextBehaviour>` ).



.. method:: PySide.QtCore.QXmlStreamReader.readNext()


    :rtype: :attr:`PySide.QtCore.QXmlStreamReader.TokenType`




    Reads the next token and returns its type.

    With one exception, once an :meth:`PySide.QtCore.QXmlStreamReader.error` is reported by :meth:`PySide.QtCore.QXmlStreamReader.readNext` , further reading of the XML stream is not possible. Then :meth:`PySide.QtCore.QXmlStreamReader.atEnd` returns true, :meth:`PySide.QtCore.QXmlStreamReader.hasError` returns true, and this function returns :attr:`QXmlStreamReader.Invalid<QXmlStreamReader.TokenType>` .

    The exception is when :meth:`PySide.QtCore.QXmlStreamReader.error` returns :attr:`PrematureEndOfDocumentError<QXmlStreamReader.Error>` . This error is reported when the end of an otherwise well-formed chunk of XML is reached, but the chunk doesn't represent a complete XML document. In that case, parsing *can* be resumed by calling :meth:`PySide.QtCore.QXmlStreamReader.addData` to add the next chunk of XML, when the stream is being read from a :class:`PySide.QtCore.QByteArray` , or by waiting for more data to arrive when the stream is being read from a :meth:`PySide.QtCore.QXmlStreamReader.device` .

    .. seealso::  :meth:`PySide.QtCore.QXmlStreamReader.tokenType`  :meth:`PySide.QtCore.QXmlStreamReader.tokenString` 


.. method:: PySide.QtCore.QXmlStreamReader.readNextStartElement()


    :rtype: :class:`PySide.QtCore.bool`




    Reads until the next start element within the current element. Returns true when a start element was reached. When the end element was reached, or when an error occurred, false is returned.

    The current element is the element matching the most recently parsed start element of which a matching end element has not yet been reached. When the parser has reached the end element, the current element becomes the parent element.

    This is a convenience function for when you're only concerned with parsing XML elements. The :ref:`QXmlStream Bookmarks Example<xml/streambookmarks>` makes extensive use of this function.
