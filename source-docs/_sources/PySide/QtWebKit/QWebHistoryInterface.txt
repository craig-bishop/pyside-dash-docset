Detailed Description
--------------------



    The :class:`PySide.QtWebKit.QWebHistoryInterface` class provides an interface to implement link history.

    The :class:`PySide.QtWebKit.QWebHistoryInterface` is an interface that can be used to keep track of visited links. It contains two pure virtual methods that are called by the WebKit engine: :meth:`PySide.QtWebKit.QWebHistoryInterface.addHistoryEntry` is used to add urls that have been visited to the interface, while :meth:`PySide.QtWebKit.QWebHistoryInterface.historyContains` is used to query whether the given url has been visited by the user. By default the :class:`PySide.QtWebKit.QWebHistoryInterface` is not set, so WebKit does not keep track of visited links.

    .. note::  The history tracked by :class:`PySide.QtWebKit.QWebHistoryInterface` is not specific to an instance of :class:`PySide.QtWebKit.QWebPage` but applies to all pages.

.. staticmethod:: PySide.QtWebKit.QWebHistoryInterface.defaultInterface()


    :rtype: :class:`PySide.QtWebKit.QWebHistoryInterface`




    Returns the default interface that will be used by WebKit. If no default interface has been set, Webkit will not keep track of visited links and a null pointer will be returned.

.. method:: PySide.QtWebKit.QWebHistoryInterface.historyContains(url)


    :param url: unicode
    :rtype: :class:`PySide.QtCore.bool`




    Called by the WebKit engine to query whether a certain ``url`` has been visited by the user already. Returns true if the ``url`` is part of the history of visited links; otherwise returns false.



.. staticmethod:: PySide.QtWebKit.QWebHistoryInterface.setDefaultInterface(defaultInterface)


    :param defaultInterface: :class:`PySide.QtWebKit.QWebHistoryInterface`




    Sets a new default interface, ``defaultInterface`` , that will be used by all of WebKit to keep track of visited links.

    If an interface without a parent has already been set, the old interface will be deleted. When the application exists :class:`PySide.QtWebKit.QWebHistoryInterface` will automatically delete the ``defaultInterface`` if it does not have a parent.
