Detailed Description
--------------------



    The :class:`PySide.QtNetwork.QNetworkRequest` class holds a request to be sent with :class:`PySide.QtNetwork.QNetworkAccessManager` .

    :class:`PySide.QtNetwork.QNetworkRequest` is part of the Network Access API and is the class holding the information necessary to send a request over the network. It contains a URL and some ancillary information that can be used to modify the request.

    .. seealso::  :class:`PySide.QtNetwork.QNetworkReply`  :class:`PySide.QtNetwork.QNetworkAccessManager`

.. attribute:: PySide.QtNetwork.QNetworkRequest.CacheLoadControl

    Controls the caching mechanism of :class:`PySide.QtNetwork.QNetworkAccessManager` .

    +-----------------------------+---------------------------------------------------------------------------------------------------------------------------------------+
    |Constant                     |Description                                                                                                                            |
    +=============================+=======================================================================================================================================+
    |QNetworkRequest.AlwaysNetwork|always load from network and do not check if the cache has a valid entry (similar to the "Reload" feature in browsers)                 |
    +-----------------------------+---------------------------------------------------------------------------------------------------------------------------------------+
    |QNetworkRequest.PreferNetwork|default value; load from the network if the cached entry is older than the network entry                                               |
    +-----------------------------+---------------------------------------------------------------------------------------------------------------------------------------+
    |QNetworkRequest.PreferCache  |load from cache if available, otherwise load from network. Note that this can return possibly stale (but not expired) items from cache.|
    +-----------------------------+---------------------------------------------------------------------------------------------------------------------------------------+
    |QNetworkRequest.AlwaysCache  |only load from cache, indicating error if the item was not cached (i.e., off-line mode)                                                |
    +-----------------------------+---------------------------------------------------------------------------------------------------------------------------------------+

.. method:: PySide.QtNetwork.QNetworkRequest.setRawHeader(headerName, value)


    :param headerName: :class:`PySide.QtCore.QByteArray`
    :param value: :class:`PySide.QtCore.QByteArray`




    Sets the header ``headerName`` to be of value ``headerValue`` . If ``headerName`` corresponds to a known header (see :attr:`QNetworkRequest.KnownHeaders` ), the raw format will be parsed and the corresponding "cooked" header will be set as well.

    For example:

    ::

        request.setRawHeader("Last-Modified", "Sun, 06 Nov 1994 08:49:37 GMT")


    will also set the known header :attr:`LastModifiedHeader<QNetworkRequest.KnownHeaders>` to be the :class:`PySide.QtCore.QDateTime` object of the parsed date.

    Note: setting the same header twice overrides the previous setting. To accomplish the behaviour of multiple HTTP headers of the same name, you should concatenate the two values, separating them with a comma (",") and set one single raw header.

    .. seealso::  :attr:`QNetworkRequest.KnownHeaders`  :meth:`PySide.QtNetwork.QNetworkRequest.setHeader`  :meth:`PySide.QtNetwork.QNetworkRequest.hasRawHeader`  :meth:`PySide.QtNetwork.QNetworkRequest.rawHeader` 


.. method:: PySide.QtNetwork.QNetworkRequest.setSslConfiguration(configuration)


    :param configuration: :class:`PySide.QtNetwork.QSslConfiguration`




    Sets this network request's SSL configuration to be ``config`` . The settings that apply are the private key, the local certificate, the SSL protocol (SSLv2, SSLv3, TLSv1 where applicable) and the ciphers that the SSL backend is allowed to use.

    By default, no SSL configuration is set, which allows the backends to choose freely what configuration is best for them.
